Suggested structure

Keep sections with ids and clear headings. The shared shell reads those sections and automatically builds the drawer navigation and command palette entries for the page.

Hero

Sections

Code blocks

Shell snippet to keep

When you copy this template into notes/, these relative paths still work, because notes/ and templates/ are both siblings of assets/. 

<link rel="stylesheet" href="../assets/css/shell.css">
<link rel="stylesheet" href="../assets/css/note-template.css">

<script src="../assets/js/notes-manifest.js" defer></script>
<script src="../assets/js/notes-data.js" defer></script>
<script src="../assets/js/site-shell.js" defer data-root=".."></script>

Quick checklist

Before you add a new note, this is the easiest path:

Copy this file into notes/ and rename it.
Replace the demo sections with your actual note content.
Add a manifest entry later if you want richer homepage metadata.
