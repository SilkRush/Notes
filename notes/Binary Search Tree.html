What is a BST?

A Binary Search Tree is a binary tree where every node obeys one rule: all values in the left subtree are strictly less than the node, and all values in the right subtree are strictly greater. That single constraint is what makes everything efficient.

Because of it, you can make a binary decision at every node — go left or go right — and eliminate half the remaining tree each time. That's why search works in O(log n) on a balanced tree. It's the same reason binary search on a sorted array works. The BST just encodes that sorted structure into a tree shape.

The Core Property

Left child (smaller)

✓ Valid BST

Every node's left subtree contains only values < node. Every node's right subtree contains only values > node. This applies recursively at every level, not just between parent and child.

✗ Common Mistake

Checking only parent-child pairs isn't enough. A node must be less than ALL ancestors on its right path and greater than ALL ancestors on its left path — that's what the validation algorithm checks.

Node Structure & Tree Anatomy

Each node holds a value and two pointers — left and right. That's it. The tree itself is often represented by just a pointer to the root node. Everything else is a property that emerges from the shape of the connections.

03

Insert

Inserting is just a guided descent. Start at the root. At each node, compare your value — if it's less, go left; if it's greater, go right. Keep going until you fall off the tree. That's where your new node goes.

The key insight: the path you take determines the exact position where the node must land to preserve the BST property. You don't choose the position — the tree tells you where it goes.

Node* insert(Node* root, int val) {
  if (root == nullptr)
    return new Node(val); // found the spot

  if (val < root->data)
    root->left = insert(root->left, val);
  else if (val > root->data)
    root->right = insert(root->right, val);
  // duplicates ignored
  return root;
}

Search

Search mirrors insert exactly — you follow the same comparison logic down the tree. The difference is you're not placing anything; you're asking "is this value here?" If you reach null, the answer is no.

On a balanced tree, each comparison cuts the remaining candidates in half. That's O(log n). On a degenerate tree (like inserting values in sorted order), every node has only one child and it degrades to O(n) — essentially a linked list.

Node* search(Node* root, int val) {
  if (root == nullptr || root->data == val)
    return root; // null = not found, non-null = found

  if (val < root->data)
    return search(root->left, val);
  return search(root->right, val);
}

05

Delete (3 Cases)

Deletion is the trickiest operation. First you search for the node. Then you handle one of three cases depending on how many children it has. Get the case wrong and you either lose part of the tree or break the BST property.

Case 1 — Leaf node

Node has no children. Just remove it. Set the parent's pointer to null. Simplest case.

Case 2 — One child

Node has one child. Bypass the node — connect the parent directly to its one grandchild. The subtree slides up by one level.

Case 3 — Two children (hardest)

You can't just remove it — the node is holding two subtrees together. The solution: replace the node's value with its inorder successor (the smallest value in the right subtree), then delete that successor from the right subtree. The successor either has no children or one right child, so deleting it is Case 1 or 2. The BST property is preserved because the successor was the smallest value greater than the deleted node.

Node* deleteNode(Node* root, int val) {
  if (!root) return nullptr;
  if (val < root->data)
    root->left = deleteNode(root->left, val);
  else if (val > root->data)
    root->right = deleteNode(root->right, val);
  else {
    if (!root->left) { // Case 1 or 2 (no left)
      Node* tmp = root->right;
      delete root; return tmp;
    }
    if (!root->right) { // Case 2 (no right)
      Node* tmp = root->left;
      delete root; return tmp;
    }
    // Case 3: find inorder successor
    Node* succ = findMin(root->right);
    root->data = succ->data;
    root->right = deleteNode(root->right, succ->data);
  }
  return root;
}

Inorder Traversal

Inorder visits nodes in the pattern: Left → Root → Right. For a BST, this always produces values in sorted ascending order. That's not a coincidence — it's a direct consequence of the BST property. If you want to extract all values sorted, inorder is how you do it.

void inorder(Node* root) {
  if (!root) return;
  inorder(root->left); // 1. Left
  cout << root->data; // 2. Root
  inorder(root->right); // 3. Right
}

Preorder Traversal

Preorder visits: Root → Left → Right. The root comes first, hence "pre." Useful for serializing a tree — if you save nodes in preorder and re-insert them, you get the same tree structure back. Also used for copying trees and printing expression trees.

void preorder(Node* root) {
  if (!root) return;
  cout << root->data; // 1. Root (first!)
  preorder(root->left); // 2. Left
  preorder(root->right); // 3. Right
}

Postorder Traversal

Postorder visits: Left → Right → Root. Root comes last. This is the right order for deleting a tree — you delete children before their parent, so you never lose a reference to something you still need. Also used for evaluating expression trees bottom-up.

void postorder(Node* root) {
  if (!root) return;
  postorder(root->left); // 1. Left
  postorder(root->right); // 2. Right
  cout << root->data; // 3. Root (last!)
}

Level-Order Traversal (BFS)

Level-order visits all nodes level by level, left to right. It uses a queue instead of recursion. You add the root, then process it, adding its children. Those children get processed next, adding their children. Width before depth.

This is breadth-first search applied to trees. It's used when you need the shortest path, or when you want to print a tree level by level, or when you need to find the minimum depth to a leaf.

void levelOrder(Node* root) {
  if (!root) return;
  queue<Node*> q;
  q.push(root);
  while (!q.empty()) {
    Node* cur = q.front(); q.pop();
    cout << cur->data;
    if (cur->left) q.push(cur->left);
    if (cur->right) q.push(cur->right);
  }
}

Min & Max

The minimum is always the leftmost node. The maximum is always the rightmost. You don't search — you just follow pointers in one direction until you can't go any further. Both are O(h) where h is height.

findMin

Node* findMin(Node* root) {
  while (root->left)
    root = root->left;
  return root;
}

findMax

Node* findMax(Node* root) {
  while (root->right)
    root = root->right;
  return root;
}

Height & Depth

Height is measured from a node downward — the number of edges on the longest path to a leaf. The height of the entire tree is the height of the root. An empty tree has height -1. A single node has height 0.

Depth is measured from a node upward to the root. Root has depth 0. Every child's depth is parent depth + 1.

int height(Node* root) {
  if (!root) return -1; // empty tree
  int lh = height(root->left);
  int rh = height(root->right);
  return 1 + max(lh, rh);
}

Predecessor & Successor

The inorder successor of a node is the next value in sorted order — the smallest value greater than it. The inorder predecessor is the previous — the largest value less than it.

Successor Logic

If the node has a right subtree, successor = minimum of that subtree. If no right subtree, the successor is the deepest ancestor for which you went left on the path from root.

Node* successor(Node* root, int val) {
  Node* succ = nullptr;
  while (root) {
    if (val < root->data)
      { succ = root; root = root->left; }
    else if (val > root->data)
      root = root->right;
    else {
      if (root->right)
        succ = findMin(root->right);
      break;
    }
  }
  return succ;
}

Predecessor Logic

Mirror of successor. If node has a left subtree, predecessor = maximum of that subtree. Otherwise, it's the deepest ancestor for which you went right.

Node* predecessor(Node* root, int val) {
  Node* pred = nullptr;
  while (root) {
    if (val > root->data)
      { pred = root; root = root->right; }
    else if (val < root->data)
      root = root->left;
    else {
      if (root->left)
        pred = findMax(root->left);
      break;
    }
  }
  return pred;
}

Validate BST

The naive approach — checking each node is greater than its left child and less than its right child — doesn't work. It misses violations that span multiple levels. The correct approach tracks a valid range [min, max] for each node. When you go right, the minimum bound becomes the parent's value. When you go left, the maximum bound becomes the parent's value.

bool isValidBST(Node* root,
                int minVal = INT_MIN,
                int maxVal = INT_MAX) {
  if (!root) return true;
  if (root->data <= minVal ||
      root->data >= maxVal)
    return false; // out of valid range
  return isValidBST(root->left, minVal, root->data) &&
         isValidBST(root->right, root->data, maxVal);
}
