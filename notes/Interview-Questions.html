OOP Concepts
0 questions
You've finished OOP — good. These questions test whether you actually understand the ideas, not just the syntax. Focus on why each concept exists, not just what it does.
01
What are the four pillars of OOP?
TechnicalEasy
▾
Answer

Encapsulation — Bundling data and the methods that work on that data into one unit (a class), and controlling access through access modifiers. The idea is to hide internal state from the outside world.

Abstraction — Hiding implementation details and exposing only what's necessary. A user of a class shouldn't need to know how it works internally, just what it does.

Inheritance — A class (child/derived) can inherit attributes and behaviors from another class (parent/base). Promotes code reuse.

Polymorphism — "Many forms." The same interface can behave differently depending on the object type. Function overloading (compile-time) and virtual functions/overriding (runtime) are both forms of polymorphism.

Interview tipThey'll often ask you to give an example of each. Have a simple, concrete example ready — like a Shape base class with Circle and Rectangle derived classes.
02
What is the difference between a class and an object?
TechnicalEasy
▾
Answer

A class is a blueprint or template. It defines the structure and behavior but doesn't occupy memory on its own in any meaningful way.

An object is an instance of a class. When you do Dog myDog;, you're creating an object — actual memory is allocated, and it has real state.

Analogy: the class is the cookie cutter, the object is the actual cookie.

class Dog {       // This is the class (blueprint)
public:
    string name;
    void bark() { cout << "Woof!"; }
};

Dog myDog;         // This is an object (instance)
myDog.name = "Rex";
myDog.bark();
03
What is inheritance and what types exist?
TechnicalMedium
▾
Answer

Inheritance lets a derived class acquire the properties and methods of a base class, promoting code reuse and establishing an "is-a" relationship.

Types in C++: single (one parent), multiple (multiple parents — can cause the Diamond Problem), multilevel (A → B → C), hierarchical (one parent, multiple children), hybrid (combination).

Diamond problemWhen class D inherits from B and C, both of which inherit from A — there's ambiguity about which A's data to use. C++ solves this with virtual inheritance.
class Animal {
public:
    void eat() { cout << "Eating"; }
};

class Dog : public Animal { // Dog IS-A Animal
public:
    void bark() { cout << "Woof"; }
};
04
What is polymorphism? Explain runtime vs compile-time polymorphism.
TechnicalMedium
▾
Answer

Compile-time (static) polymorphism is resolved at compile time. Examples: function overloading and operator overloading. The compiler knows which version of the function to call based on the argument types.

Runtime (dynamic) polymorphism is resolved at runtime. This is what virtual functions do. A base class pointer/reference can call the correct overridden method in a derived class — the decision happens while the program is running.

class Shape {
public:
    virtual void draw() { cout << "Shape"; }
};
class Circle : public Shape {
public:
    void draw() override { cout << "Circle"; }
};
Shape* s = new Circle();
s->draw(); // Prints "Circle" — runtime dispatch via vtable
vtableThe compiler generates a virtual function table (vtable) for classes with virtual functions. Each object stores a pointer to its vtable. That's how the correct function gets called at runtime.
05
What is encapsulation and why does it matter?
TechnicalEasy
▾
Answer

Encapsulation is about bundling data (member variables) and the functions that operate on that data into a single unit (a class), and then restricting direct access to that data from outside the class.

Why does it matter? If external code can directly modify internal state, any part of the program could corrupt that state. Encapsulation forces all access to go through methods you control, so you can validate data, enforce invariants, and change the internal implementation without breaking outside code.

class BankAccount {
private:
    double balance;    // Hidden — can't set directly
public:
    void deposit(double amount) {
        if (amount > 0) balance += amount; // Validated
    }
    double getBalance() { return balance; }
};
06
What is the Rule of Three in C++?
TechnicalMedium
▾
Answer

If your class manages a resource (heap memory, file handle, network connection), you need to explicitly define all three of: destructor, copy constructor, and copy assignment operator.

If you only define one, the compiler-generated versions of the others will do a shallow copy — copying the pointer, not what it points to. Two objects end up pointing to the same memory. When both destructors run, you get a double-free crash.

Rule of Five (C++11+)Extends Rule of Three by also adding move constructor and move assignment operator. If you're managing resources, handle all five.
Best practiceUse smart pointers (unique_ptr, shared_ptr) whenever possible — they handle this automatically.
07
What is an abstract class? How is it different from an interface?
TechnicalMedium
▾
Answer

An abstract class has at least one pure virtual function (virtual void foo() = 0;). You can't instantiate it directly — it's a base class meant to be derived from. It can have both implemented methods and pure virtual ones.

An interface (in languages like Java/C#) is a contract with only pure virtual methods and no implementation at all. In C++, you simulate interfaces with abstract classes that have only pure virtual functions.

class Animal {         // Abstract class
public:
    virtual void speak() = 0;  // Pure virtual
    void breathe() { cout << "breathing"; } // Concrete
};
// Animal a; // ERROR — can't instantiate
class Dog : public Animal {
public:
    void speak() override { cout << "Woof"; }
};

01
What is Big O notation? Why does it matter?
TechnicalEasy
▾
Answer

Big O notation describes how the runtime (or space) of an algorithm scales as the input size n grows. It ignores constants and focuses on the dominant term — the worst-case behavior.

Common Complexities (best → worst)
O(1)Constant
O(log n)Logarithmic
O(n)Linear
O(n log n)Linearithmic
O(n²)Quadratic
O(2ⁿ)Exponential
Interview tipAfter you code a solution, always say its time and space complexity unprompted. It shows you're thinking about performance, not just correctness.
02
What is the difference between an array and a linked list?
TechnicalEasy
▾
Answer
Array
AccessO(1)
SearchO(n)
Insert/DeleteO(n)
MemoryContiguous
Linked List
AccessO(n)
SearchO(n)
Insert/DeleteO(1) at head

Use an array when you need fast random access. Use a linked list when you're doing a lot of insertions and deletions and don't need to index into it constantly.

Cache performanceArrays are cache-friendly because their elements are contiguous in memory. Linked lists are not — each node is a separate heap allocation, causing cache misses. This makes arrays faster in practice even when theoretical complexity is the same.
03
What is a stack? What is a queue? Give real-world examples.
TechnicalEasy
▾
Answer

A stack is LIFO — Last In, First Out. The last thing you pushed is the first thing you get back. Think of a stack of plates. Key operations: push, pop, peek/top. All O(1).

Real-world uses: function call stack (each function call pushes a frame), undo/redo in a text editor, expression evaluation (matching parentheses), backtracking in DFS.

A queue is FIFO — First In, First Out. Like a line at a store. Key operations: enqueue, dequeue. All O(1).

Real-world uses: task scheduling, BFS graph traversal, print job queuing, message buffers.

04
What is a Binary Search Tree (BST)? What are its properties?
TechnicalMedium
▾
Answer

A BST is a binary tree where every node satisfies: all nodes in the left subtree have smaller values and all nodes in the right subtree have larger values.

Search: O(log n) avg, O(n) worst Insert: O(log n) avg Delete: O(log n) avg

The worst case O(n) happens when the tree is degenerate — basically a linked list (e.g., inserting sorted data into a basic BST). This is why balanced BSTs like AVL trees and Red-Black trees exist.

Interview tipKnow in-order, pre-order, and post-order traversal. In-order traversal of a BST always gives you sorted output — that's a very common interview observation.
05
What is a hash table? How does it handle collisions?
TechnicalMedium
▾
Answer

A hash table maps keys to values using a hash function that converts the key to an index in an array. Lookup, insertion, and deletion are all O(1) on average — that's why it's so powerful.

A collision is when two different keys hash to the same index. Two main strategies:

Chaining — Each slot in the array holds a linked list of all key-value pairs that hash there. Simple, handles many collisions gracefully. Worst case O(n) if all keys hash to the same slot.

Open addressing — When a collision occurs, probe for the next available slot (linear probing, quadratic probing, or double hashing). All data stays in the array itself.

Load factorWhen too many entries are in the table relative to its size, performance degrades. A good hash table resizes (rehashes) when the load factor exceeds a threshold (~0.7).
06
What is recursion? When would you use it over a loop?
TechnicalEasy
▾
Answer

Recursion is when a function calls itself to solve a smaller version of the same problem, until it hits a base case (a condition where it stops).

Every recursive solution needs two things: a base case (stops the recursion) and a recursive case (breaks the problem down).

Use recursion when the problem has a naturally recursive structure — trees, graphs, divide-and-conquer algorithms (merge sort, binary search), and problems with multiple subproblems of the same type.

Stack overflow riskEach recursive call adds a frame to the call stack. Very deep recursion on large inputs can overflow the stack. Iterative solutions with an explicit stack are sometimes safer for large inputs.
Fibonacci exampleNaive recursive Fibonacci is O(2ⁿ) — terrible. With memoization (Dynamic Programming), it becomes O(n). Know this distinction.
07
What is the difference between BFS and DFS?
TechnicalMedium
▾
Answer

BFS (Breadth-First Search) explores level by level, using a queue. It finds the shortest path in an unweighted graph. Good when you need to find something close to the source.

DFS (Depth-First Search) goes as deep as possible down one path before backtracking, using a stack (or recursion). Good for exploring all paths, cycle detection, topological sorting, connected components.

These are the fundamentals that show up in internship interviews even when the role isn't specifically systems-focused. Know them well enough to explain them clearly.
01
What is the difference between a process and a thread?
TechnicalMedium
▾
Answer

A process is an independent program in execution with its own memory space (heap, stack, code segment). Processes are isolated — they can't directly access each other's memory. Communication requires IPC (inter-process communication).

A thread is a lightweight unit of execution within a process. Threads in the same process share memory (heap, global variables) but each has its own stack and registers. Creating a thread is cheaper than creating a process.

Key implicationBecause threads share memory, they can communicate faster but are prone to race conditions. Processes are more isolated and safer but slower to spin up and communicate.
02
What is memory management? What are stack vs heap?
TechnicalMedium
▾
Answer

The stack is used for static memory allocation — local variables and function call frames. Memory is allocated and freed automatically as functions are called and return. It's fast and small. You can't control its size at runtime.

The heap is used for dynamic memory allocation — new/malloc in C++. You control when memory is allocated and freed. It's larger but slower and fragmented. You're responsible for freeing it (delete/free) — forgetting causes memory leaks.

Memory leakHeap memory that was allocated but never freed. In long-running programs, this gradually eats all available memory and causes crashes.
RAII in C++Resource Acquisition Is Initialization — tie resource lifetime to object lifetime. Smart pointers (unique_ptr) do this automatically. Learn this pattern.
03
What is a pointer? What is a reference? How are they different?
TechnicalMedium
▾
Answer

A pointer stores the memory address of another variable. It can be null, reassigned to point to something else, and can do pointer arithmetic.

A reference is an alias — another name for an existing variable. It must be initialized when declared, can't be null, and can't be made to refer to something else after initialization.

int x = 10;
int* ptr = &x;   // pointer — stores address of x
int& ref = x;   // reference — alias for x

*ptr = 20;      // x is now 20 (dereference)
ref = 30;       // x is now 30 (direct)
When to use whichUse references when you don't need to change what you're pointing at (function parameters). Use pointers when you need nullptr, dynamic allocation, or rebinding.
04
What is the difference between TCP and UDP?
TechnicalEasy
▾
Answer

TCP (Transmission Control Protocol) is connection-oriented. It guarantees delivery, ordering, and error-checking. Uses a handshake. Slower but reliable. Used for: HTTP/HTTPS, email, file transfers — anything where losing a packet is unacceptable.

UDP (User Datagram Protocol) is connectionless. It fires packets and forgets them — no guarantee of delivery or order. Faster, lower overhead. Used for: video streaming, online gaming, DNS, live calls — speed matters more than perfection.

AnalogyTCP is like certified mail — guaranteed delivery, acknowledgment. UDP is like dropping flyers from a plane — fast, but some might get lost and nobody confirms receipt.
05
What happens when you type a URL in your browser and press Enter?
TechnicalMedium
▾
Answer

A classic interview question. Walk through it step by step:

DNS lookup — Browser checks cache, then OS, then DNS resolver to get the IP address for the domain.
TCP connection — Browser establishes a TCP connection with the server using a 3-way handshake (SYN → SYN-ACK → ACK).
TLS handshake — If HTTPS, a TLS handshake happens to establish encryption.
HTTP request — Browser sends an HTTP GET request for the resource.
Server response — Server processes and sends back HTML, CSS, JS, etc.
Rendering — Browser parses HTML → builds DOM → fetches linked resources → renders the page.
Interview tipYou don't have to memorize every detail. Showing you understand the general flow (DNS → TCP → HTTP → render) is what they're looking for. Extra detail earns extra points.

Coding Challenges
0 questions
These are the patterns that show up most frequently in internship-level coding interviews. For each one: talk through your approach first, code it, then analyze complexity. Don't jump straight to code in silence.
01
Reverse a string (without using a built-in reverse function).
TechnicalEasy
▾
Answer

Two-pointer approach: one pointer at the start, one at the end. Swap characters and move both inward until they meet.

void reverseString(string& s) {
    int left = 0, right = s.size() - 1;
    while (left < right) {
        swap(s[left], s[right]);
        left++;
        right--;
    }
}
Time: O(n) Space: O(1)
Talk through itSay: "I'll use two pointers starting at the ends, swap them, and move toward the middle. This is O(n) time and O(1) space since I'm doing it in-place."
02
Check if a string is a palindrome.
TechnicalEasy
▾
Answer

A palindrome reads the same forwards and backwards. Two-pointer approach: compare first and last characters, move inward. If any pair doesn't match, it's not a palindrome.

bool isPalindrome(const string& s) {
    int left = 0, right = s.size() - 1;
    while (left < right) {
        if (s[left] != s[right]) return false;
        left++;
        right--;
    }
    return true;
}
Time: O(n) Space: O(1)
Follow-upInterviewers often ask: what about "A man a plan a canal Panama"? You'd need to ignore spaces, punctuation, and case. Handle with a preprocessing step or skip non-alphanumeric characters in the loop.
03
Find the two numbers in an array that add up to a target sum.
TechnicalEasy
▾
Answer

Brute force is O(n²) — check every pair. The optimal solution uses a hash set: for each number, check if its complement (target - num) is already in the set.

// Returns indices of the two numbers
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int,int> seen; // value → index
    for (int i = 0; i < nums.size(); i++) {
        int complement = target - nums[i];
        if (seen.count(complement))
            return {seen[complement], i};
        seen[nums[i]] = i;
    }
    return {};
}
Time: O(n) Space: O(n)
Pattern to knowWhenever you're trading space for time, think hash map. This "complement lookup" pattern appears in dozens of interview questions.
04
Find the maximum subarray sum (Kadane's Algorithm).
TechnicalMedium
▾
Answer

Given an array, find the contiguous subarray with the largest sum. Kadane's Algorithm does this in O(n) by tracking the current running sum and resetting it when it goes negative.

int maxSubArray(vector<int>& nums) {
    int maxSum = nums[0];
    int current = nums[0];
    for (int i = 1; i < nums.size(); i++) {
        // Either extend current subarray or start fresh
        current = max(nums[i], current + nums[i]);
        maxSum = max(maxSum, current);
    }
    return maxSum;
}
Time: O(n) Space: O(1)
Why it worksIf adding the current element to a negative running sum makes it worse than just taking the current element alone, start a new subarray. Otherwise, extend.
05
Check if a binary tree is a valid BST.
TechnicalMedium
▾
Answer

A common mistake is only checking left child < node and right child > node. That's not enough — the entire left subtree must be less, entire right subtree must be greater. Pass valid ranges down recursively.

bool isValidBST(TreeNode* node, long minVal, long maxVal) {
    if (!node) return true;
    if (node->val <= minVal || node->val >= maxVal)
        return false;
    return isValidBST(node->left,  minVal,      node->val) &&
           isValidBST(node->right, node->val, maxVal);
}
// Call with: isValidBST(root, LONG_MIN, LONG_MAX)
Time: O(n) Space: O(h) (call stack)
Common mistakeOnly comparing a node to its direct children. You need to enforce the full range constraint all the way down each branch.
06
Find the first non-repeating character in a string.
TechnicalEasy
▾
Answer

Two-pass approach: first count frequencies of all characters using a hash map. Second pass through the original string — return the first character with frequency 1.

int firstUniqChar(string s) {
    unordered_map<char,int> freq;
    for (char c : s) freq[c]++;
    for (int i = 0; i < s.size(); i++)
        if (freq[s[i]] == 1) return i;
    return -1; // no unique char found
}
Time: O(n) Space: O(1) (26 letters max)
Space noteEven though we use a map, the number of distinct characters is bounded by the alphabet size (26 or 128 for ASCII), so space is effectively O(1).
07
Check if two strings are anagrams of each other.
TechnicalEasy
▾
Answer

Two strings are anagrams if they contain the exact same characters in the same frequencies. Count frequencies of each string and compare. Or sort both and compare — though sorting is O(n log n).

bool isAnagram(string s, string t) {
    if (s.size() != t.size()) return false;
    int count[26] = {0};
    for (int i = 0; i < s.size(); i++) {
        count[s[i] - 'a']++;
        count[t[i] - 'a']--;
    }
    for (int c : count)
        if (c != 0) return false;
    return true;
}
Time: O(n) Space: O(1)
08
Find the missing number in an array of 0 to n.
TechnicalEasy
▾
Answer

The expected sum of 0 to n is n*(n+1)/2. Compute the actual sum of the array. The difference is the missing number. No extra space needed.

int missingNumber(vector<int>& nums) {
    int n = nums.size();
    int expected = n * (n + 1) / 2;
    int actual = 0;
    for (int x : nums) actual += x;
    return expected - actual;
}
Time: O(n) Space: O(1)
XOR trickAnother approach: XOR all indices and all values. Missing number is what's left. Both approaches are O(n) / O(1). The math approach is easier to explain.
09
Valid parentheses — check if bracket sequence is balanced.
TechnicalEasy
▾
Answer

Classic stack problem. Push opening brackets onto the stack. When you see a closing bracket, check if the top of the stack is the matching opener. If it doesn't match or the stack is empty, return false. At the end the stack must be empty.

bool isValid(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{') {
            st.push(c);
        } else {
            if (st.empty()) return false;
            char top = st.top(); st.pop();
            if (c == ')' && top != '(') return false;
            if (c == ']' && top != '[') return false;
            if (c == '}' && top != '{') return false;
        }
    }
    return st.empty();
}
Time: O(n) Space: O(n)
10
Merge two sorted arrays into one sorted array.
TechnicalEasy
▾
Answer

Two-pointer merge: pointer i on first array, pointer j on second. Compare elements, take the smaller one, advance that pointer. After one array is exhausted, append the rest of the other.

vector<int> mergeSorted(vector<int>& a, vector<int>& b) {
    vector<int> result;
    int i = 0, j = 0;
    while (i < a.size() && j < b.size()) {
        if (a[i] <= b[j]) result.push_back(a[i++]);
        else              result.push_back(b[j++]);
    }
    while (i < a.size()) result.push_back(a[i++]);
    while (j < b.size()) result.push_back(b[j++]);
    return result;
}
Time: O(m+n) Space: O(m+n)
This is the merge stepThis exact logic is the core of Merge Sort. Knowing it well helps you understand sorting algorithms too.
11
Find the longest common prefix among an array of strings.
TechnicalEasy
▾
Answer

Take the first string as the initial prefix. Then compare it character by character against every other string, shortening the prefix whenever a mismatch is found.

string longestCommonPrefix(vector<string>& strs) {
    if (strs.empty()) return "";
    string prefix = strs[0];
    for (int i = 1; i < strs.size(); i++) {
        while (strs[i].find(prefix) != 0)
            prefix = prefix.substr(0, prefix.size() - 1);
        if (prefix.empty()) return "";
    }
    return prefix;
}
Time: O(S) — S = total chars Space: O(1)
12
Binary search — search for a target in a sorted array.
TechnicalEasy
▾
Answer

Classic divide and conquer. Maintain left and right pointers. Check the middle element — if it's the target, return. If target is larger, search right half. If smaller, search left half.

int binarySearch(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // avoid overflow
        if      (nums[mid] == target) return mid;
        else if (nums[mid] <  target) left  = mid + 1;
        else                           right = mid - 1;
    }
    return -1;
}
Time: O(log n) Space: O(1)

Implement a stack using two queues.
TechnicalMedium

Push to q1. For pop, transfer all elements of q1 to q2 except the last one — that last one is the top of the stack. Then swap q1 and q2. This simulates LIFO using FIFO queues.

    void push(int x) { q1.push(x); }

        while (q1.size() > 1) {
            q2.push(q1.front()); q1.pop();
        }
        int top = q1.front(); q1.pop();
        swap(q1, q2);
        return top;
    }
    int top()     { /* similar to pop but don't remove */ }
    bool empty()  { return q1.empty(); }

TechnicalMedium
▾
Answer

int lengthOfLongestSubstring(string s) {

    int left = 0, maxLen = 0;
    for (int right = 0; right < s.size(); right++) {
        while (window.count(s[right]))
            window.erase(s[left++]);
        window.insert(s[right]);
        maxLen = max(maxLen, right - left + 1);
    }
    return maxLen;

Product of array except self — no division allowed.
TechnicalMedium
▾
Answer

For each index i, the answer is (product of all elements to the left) × (product of all elements to the right). Do two passes: one left-to-right building prefix products, one right-to-left building suffix products.

vector<int> productExceptSelf(vector<int>& nums) {
    int n = nums.size();
    vector<int> result(n, 1);
    // Left pass: result[i] = product of all left of i
    int prefix = 1;
    for (int i = 0; i < n; i++) {
        result[i] = prefix;
        prefix *= nums[i];
    }
    // Right pass: multiply in product of all right of i
    int suffix = 1;
    for (int i = n-1; i >= 0; i--) {
        result[i] *= suffix;
        suffix *= nums[i];
    }
    return result;
}

void merge(vector<int>& arr, int l, int m, int r) {
    vector<int> left(arr.begin()+l, arr.begin()+m+1);
    vector<int> right(arr.begin()+m+1, arr.begin()+r+1);
    int i=0, j=0, k=l;
    while(i<left.size() && j<right.size())
        arr[k++] = (left[i]<=right[j]) ? left[i++] : right[j++];
    while(i<left.size())  arr[k++]=left[i++];

}
void mergeSort(vector<int>& arr, int l, int r) {
    if (l >= r) return;
    int m = l + (r - l) / 2;
    mergeSort(arr, l, m);
    mergeSort(arr, m+1, r);
    merge(arr, l, m, r);
}

TechnicalMedium

Answer

Kahn's algorithm (BFS). Compute in-degrees. Queue all nodes with in-degree 0. Process each — reduce neighbor in-degrees. Enqueue any neighbor that reaches in-degree 0.

vector<int> topoSort(int n, vector<vector<int>>& adj) {
    vector<int> inDeg(n,0), order;
    for(auto& e:adj) for(int v:e) inDeg[v]++;
    queue<int> q;
    for(int i=0;i<n;i++) if(!inDeg[i]) q.push(i);
    while(!q.empty()){
        int u=q.front(); q.pop(); order.push_back(u);
        for(int v:adj[u]) if(--inDeg[v]==0) q.push(v);
    }
    return order; // size < n means cycle exists
}

Implement an LRU Cache with O(1) get and put.
TechnicalHard
▾
Answer

Hash map + doubly linked list. Map gives O(1) lookup. List gives O(1) reordering. Most-recently-used at front, least-recently-used at back. On overflow, remove from back.

class LRUCache {
    int cap;
    list<pair<int,int>> dll;
    unordered_map<int,list<pair<int,int>>::iterator> mp;
public:
    LRUCache(int c):cap(c){}
    int get(int k){
        if(!mp.count(k)) return -1;
        dll.splice(dll.begin(),dll,mp[k]);
        return mp[k]->second;
    }
    void put(int k,int v){
        if(mp.count(k)) dll.erase(mp[k]);
        dll.push_front({k,v}); mp[k]=dll.begin();
        if(dll.size()>cap){mp.erase(dll.back().first);dll.pop_back();}
    }
};
get: O(1)put: O(1)

int partition(vector<int>& a,int lo,int hi){
    int pivot=a[hi], i=lo-1;
    for(int j=lo;j<hi;j++)
        if(a[j]<=pivot) swap(a[++i],a[j]);
    swap(a[i+1],a[hi]); return i+1;
}
void quickSort(vector<int>& a,int lo,int hi){
    if(lo<hi){ int p=partition(a,lo,hi); quickSort(a,lo,p-1); quickSort(a,p+1,hi); }
}

What Makes a Portfolio Actually Work
📁
GitHub profile matters
Pin your best 4–6 repos. Write a clear bio. Add your website link. A filled-in GitHub profile with a green contribution graph signals consistency — even if some days it's just updating a README.
📄
README = your project resume
Every repo should have a README with: what the project is, what problem it solves, how to build/run it, a screenshot or demo, and what you'd do next. No README = no one looks at the code.
🔗
Live demos > screenshots
If a project can be deployed, deploy it. A live URL you can type into a browser during an interview is worth more than any description. Free hosting exists — there's no excuse for "it only runs locally."
✅
Quality over quantity
Three complete, documented, deployed projects are better than ten that are half-built. An interviewer who clicks a link and sees a broken page thinks less of you than if you had fewer projects but all working.
🗣️
Know your projects cold
In the interview, they will ask you to explain any project on your resume. Know every design decision. Know what problems you hit and how you solved them. Know what you'd do differently. Never have a project on your resume you can't speak to deeply.
🌱
Start now, grow later
The best portfolio project is one that exists. Start with something small that works. Then improve it. "I built this in a week and have been improving it since" is a better story than "I'm planning to build something bigger."
