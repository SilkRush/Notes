01
What is Bash?
The shell, its history, and why it matters

Bash stands for Bourne Again Shell. It's a command-line interpreter — a program that reads what you type, figures out what you mean, and tells the operating system to do it. Think of it as the translator between you and the Linux kernel.

The original shell was written by Stephen Bourne in 1979. Bash came in 1989 as a free, enhanced replacement — and it's been the default shell on most Linux systems and macOS (until zsh took over on Mac) ever since.

⚡ BASH ARCHITECTURE

🖥 SHELL

Reads your commands, parses them, and communicates with the kernel. Bash is one type of shell.

📟 TERMINAL

The window/emulator you type into. Gnome Terminal, Alacritty, iTerm2 — these are terminals that run a shell.

🐧 KERNEL

The actual OS core. The shell sends your commands to it as system calls (fork, exec, open, etc.).

TIP — Check your shell
Run echo $SHELL to see which shell you're using. Run bash --version to see Bash's version.
BASH
# Check what shell you're running
echo $SHELL        # → /bin/bash
bash --version     # → GNU bash, version 5.x
which bash         # → /bin/bash  (location of bash binary)
cat /etc/shells    # → all valid shells on the system

Bash isn't just for typing commands one by one. It's a full programming language — variables, loops, functions, arrays, string manipulation, error handling, all of it. A Bash script is just a file full of commands that Bash runs in sequence.

Variables & Data Types
Storing and using data in Bash

Bash variables are untyped by default. Everything is a string, technically. But Bash can treat values as integers when you tell it to. There's no int or float like in C++.

VARIABLE BASICS
# Assigning variables — NO spaces around the = sign
name="Abhishek"
age=20
pi="3.14159"

# Reading variables — use $ to expand
echo $name         # → Abhishek
echo "Hello, $name!"   # → Hello, Abhishek!
echo "Age: ${age}"    # → Age: 20  (braces make boundary clear)

# WRONG — this will error or give wrong results
name = "Abhishek"  # ✗ spaces around = break it
CRITICAL — No spaces around =
In Bash, name="value" assigns a variable. But name = "value" means: run a command called name with arguments = and "value". The space matters. This is one of the most common beginner mistakes.
VARIABLE TYPES
# Declare types explicitly (optional but good practice)
declare -i count=10      # integer
declare -r PI="3.14"    # readonly (constant)
declare -l lower="HI"  # auto-lowercase → "hi"
declare -u upper="hi"  # auto-uppercase → "HI"
declare -a arr           # array
declare -A map           # associative array (hash map)

# Unset a variable
unset name

# Check if variable is set
if [ -z "${name}" ]; then
  echo "name is empty or not set"
fi

Quoting Rules
Single quotes, double quotes, and escaping — the source of many bugs

Quoting in Bash controls what gets expanded and what gets treated literally. Getting this wrong causes subtle bugs that are hard to find. There are three types.

💬 QUOTING COMPARISON DEMO

Set a value and compare...
Quote TypeSyntaxWhat it does
Double quotes"..."Expands variables and command substitutions, but not globs
Single quotes'...'Everything is literal. No expansion at all.
Backtick`cmd`Command substitution — old style, use $() instead
Escape\Escape the next character — treat it literally
No quotesbare wordWord splitting and glob expansion happen
QUOTING EXAMPLES
name="Abhishek"
dir="my folder"  # has a space

# Double quotes — variables expand
echo "Hello $name"       # → Hello Abhishek
echo "Today: $(date)"   # → Today: Mon Apr 13...

# Single quotes — everything literal
echo 'Hello $name'       # → Hello $name  (no expansion!)
echo '$(date)'           # → $(date)  (not evaluated)

# Why quoting variables matters
ls $dir    # ✗ splits on space → ls my folder (two args!)
ls "$dir"  # ✓ treats as one arg → ls "my folder"

# Always quote variables in paths and comparisons
if [ "$name" = "Abhishek" ]; then
  echo "Match!"
fi

# Escaping special chars
echo "Cost: \$5"     # → Cost: $5  (escaped $)
echo "Tab:\there"   # → Tab:	here (with -e flag for \t)
echo -e "Line1\nLine2"  # -e enables escape sequences
RULE OF THUMB
Always double-quote your variables: "$var". This prevents word splitting and glob expansion from causing bugs when values contain spaces or special characters. Only skip quotes when you intentionally need splitting.

Output to the terminal

ECHO

printf "%05d\n" 42               # → 00042 (zero padded)
printf "%x\n" 255               # → ff (hex)

Redirections & Pipes
Controlling where data flows — the most powerful concept in Bash

In Unix, everything is a stream. Every process has three streams: stdin (input), stdout (output), stderr (errors). You can redirect any of these. Pipes connect the stdout of one command to the stdin of another. This is how you build complex tools from simple ones.

IFS= and -r
When reading files, always use IFS= read -r line. The IFS= prevents stripping leading/trailing whitespace. The -r prevents backslash interpretation. Without these, lines with special characters can be corrupted.

Conditionals — if / elif / else
Making decisions in scripts

An if statement in Bash runs a command and checks its exit code. Zero means true (success). Non-zero means false (failure). That's it. The [ ] and [[ ]] constructs are just commands that return 0 or 1.

🔀 IF FLOW VISUALIZER

Enter a number to test
IF SYNTAX
# Basic structure
if condition; then
  # runs if condition is true (exit code 0)
elif other_condition; then
  # runs if previous was false, this is true
else
  # runs if all conditions were false
fi

# Examples
age=20

if [ $age -ge 18 ]; then
  echo "Adult"
elif [ $age -ge 13 ]; then
  echo "Teenager"
else
  echo "Child"
fi

# Short circuit — AND (&&) and OR (||)
[ $age -ge 18 ] && echo "Adult"    # run only if true
[ $age -lt 18 ] || echo "Not minor"  # run only if false

# Ternary-style one-liner
result=$([ $age -ge 18 ] && echo "adult" || echo "minor")

Test & Comparison
[ ] vs [[ ]] vs (( )) — which to use and when

There are three different ways to write conditions in Bash. They're not interchangeable. Each has different behavior and different supported operators.

STRING COMPARISONS
# [ ] — POSIX style, works in any shell
[ "$a" = "$b" ]    # equal (= not ==)
[ "$a" != "$b" ]   # not equal
[ -z "$a" ]       # empty string
[ -n "$a" ]       # non-empty string

# [[ ]] — Bash extended test (preferred for Bash scripts)
[[ "$a" == "$b" ]]     # equal
[[ "$a" == "he*" ]]     # glob pattern match
[[ "$a" =~ ^[0-9]+$ ]]  # regex match!
[[ "$a" < "$b" ]]       # lexicographic order

# (( )) — arithmetic context (integers only)
(( a > b ))
(( a == b ))
(( a % 2 == 0 ))   # even number check
NUMERIC COMPARISONS ([ ] style)
# Use -eq -ne -lt -le -gt -ge for numbers inside [ ]
[ $a -eq $b ]   # equal
[ $a -ne $b ]   # not equal
[ $a -lt $b ]   # less than
[ $a -le $b ]   # less than or equal
[ $a -gt $b ]   # greater than
[ $a -ge $b ]   # greater than or equal
FILE TESTS
[ -e file ]    # exists (any type)
[ -f file ]    # regular file
[ -d dir ]     # directory
[ -r file ]    # readable
[ -w file ]    # writable
[ -x file ]    # executable
[ -s file ]    # non-empty (size > 0)
[ -L file ]    # symbolic link
[ -p file ]    # named pipe
[ f1 -nt f2 ] # f1 newer than f2
[ f1 -ot f2 ] # f1 older than f2

case Statements
Pattern-based branching — cleaner than chained elifs

When branching on the same variable multiple times, case is far cleaner than a chain of elif blocks. It uses glob-style matching, not regex.

CASE SYNTAX
case $variable in
  pattern1)
    commands ;;
  pattern2|pattern3)
    # matches either ;;
  *.sh)
    echo "Shell script" ;;
  [0-9]*)
    echo "Starts with digit" ;;
  *)
    echo "default" ;;
esac

# Real example — service control script
case $1 in
  start|run)   echo "Starting..." ;;
  stop)        echo "Stopping..." ;;
  restart)     echo "Restarting..." ;;
  --help|-h)   echo "Usage: $0 {start|stop|restart}" ;;
  *)           echo "Unknown: $1"; exit 1 ;;
esac

Loop Control — break & continue
Early exit and iteration skipping
BREAK & CONTINUE
# break — exit loop entirely
for i in {1..10}; do
  [ $i -eq 5 ] && { echo "Stop at 5"; break; }
  echo $i
done   # prints 1 2 3 4 Stop at 5

# continue — skip this iteration
for i in {1..10}; do
  (( i % 2 == 0 )) && continue
  echo $i   # prints 1 3 5 7 9
done

# break N — break out of N nested loops
for (( i=0; i<5; i++ )); do
  for (( j=0; j<5; j++ )); do
    (( i*j > 6 )) && break 2

  done
done

Functions
Reusable named command blocks

Functions in Bash don't have typed parameters. Arguments are passed exactly like script arguments — $1, $2, etc. Define before you call.

FUNCTIONS
# Two valid declaration styles
function greet() { echo "Hello, $1!"; }
greet() { echo "Hello, $1!"; }   # POSIX-compatible (preferred)

greet "Abhishek"   # → Hello, Abhishek!

# Multiple args
add_user() {
  local username="$1"
  local role="\${2:-viewer}"   # default role
  echo "Adding '$username' as '$role'"
}
add_user "abhishek" "admin"   # → Adding 'abhishek' as 'admin'
add_user "bob"                # → Adding 'bob' as 'viewer'

# "Return" a value via stdout
get_square() { echo $(( $1 * $1 )); }
result=$(get_square 7)   # → 49

# Return exit code
is_even() { (( $1 % 2 == 0 )); }
is_even 4 && echo "even" || echo "odd"
DEFINE BEFORE CALL
Bash reads top to bottom. A function must be defined before the line that calls it. Unlike C++, there are no forward declarations.

SCOPE & RETURN

get_stats() {

STRING OPERATIONS

Word splitting is what Bash does after it expands a variable: it chops the result into separate words using IFS (Internal Field Separator) as the knife. If you don't understand this, you will write broken code forever.

COMPLETE GETOPTS PATTERN
#!/usr/bin/env bash
set -euo pipefail

# Usage function
usage() {
  cat <<EOF
Usage: $(basename "$0") [OPTIONS] <input_file>

Options:
  -o <file>   Output file (default: output.txt)
  -n <num>    Number of lines (default: 10)
  -v          Verbose mode
  -f          Force overwrite
  -h          Show this help

Examples:
  $(basename "$0") -v -n 20 -o result.txt data.csv
  $(basename "$0") -f input.txt
EOF
  exit 0
}

# Defaults
output="output.txt"
lines=10
verbose=false
force=false

# Parse options
# Leading : = silent mode (we handle errors ourselves)
while getopts ":o:n:vfh" opt; do
  case $opt in
    o) output="$OPTARG" ;;
    n) lines="$OPTARG"
       # Validate it's a number
       [[ "$lines" =~ ^[0-9]+$ ]] || { echo "ERROR: -n needs a number"; exit 1; } ;;
    v) verbose=true ;;
    f) force=true ;;
    h) usage ;;
    :) echo "ERROR: -$OPTARG requires an argument"; exit 1 ;;
    ?) echo "ERROR: Unknown option -$OPTARG"; usage ;;
  esac

# ── RULE 1: Avoid subshells in loops ──
# SLOW — runs a new process for every iteration
for f in *.txt; do

🧠 MNEMONIC — Bash does 8 steps in order: "BTPAG WPS"

#!/usr/bin/env bash
set -euo pipefail

# Thresholds
CPU_WARN=80; MEM_WARN=85; DISK_WARN=90

# Status helpers
OK()   { printf '\033[32m ✓ %-30s %s\033[0m\n' "$1" "$2"; }
WARN() { printf '\033[33m ⚠ %-30s %s\033[0m\n' "$1" "$2"; }
FAIL() { printf '\033[31m ✗ %-30s %s\033[0m\n' "$1" "$2"; }

echo -e "\n\033[1;32m═══ SYSTEM HEALTH CHECK — $(date) ═══\033[0m\n"

# ── CPU LOAD ──
cpu_idle=$(top -bn1 | grep "Cpu(s)" | awk '{print $8}' | tr -d '%')
cpu_use=$(( 100 - ${cpu_idle%.*} ))
if   (( cpu_use >= CPU_WARN )); then FAIL "CPU Usage" "${cpu_use}%"
else OK "CPU Usage" "${cpu_use}%"; fi

# ── MEMORY ──
read -r total used free <<< $(free -m | awk 'NR==2{print $2,$3,$4}')
mem_pct=$(( used * 100 / total ))
(( mem_pct >= MEM_WARN )) && WARN "Memory" "${mem_pct}% (${used}/${total} MB)" \
                           || OK   "Memory" "${mem_pct}% (${used}/${total} MB)"

# ── DISK ──
while IFS= read -r line; do
  pct=${line%%%*}; mount=$(echo "$line" | awk '{print $NF}')
  (( pct >= DISK_WARN )) && FAIL "Disk $mount" "${pct}% used" \
                           || OK   "Disk $mount" "${pct}% used"
done < <(df -h | awk 'NR>1 && /\// {gsub(/%/,"",$5); print $5, $6}')

# ── SERVICES ──
for svc in ssh nginx mysql; do
  systemctl is-active --quiet "$svc" 2>/dev/null \
    && OK "Service $svc" "running" \
    || FAIL "Service $svc" "NOT running"
done

# ── UPTIME ──
OK "Uptime" "$(uptime -p)"

echo -e "\n\033[32m═══════════════════════════════════\033[0m"

TOP PITFALLS WITH FIXES
# 1. Space around = in assignment
name = "Abhishek"   # ✗ runs command 'name' with args
name="Abhishek"    # ✓

# 2. Unquoted variables
[ $var = "x" ]      # ✗ fails if $var is empty
[ "$var" = "x" ]   # ✓

for f in *          # ✓ (glob directly)

cd "$dir" || exit 1; rm *   # ✓

result=$(echo "hello\n")    # $() strips trailing newlines — expected behavior

# 6. Comparing integers with = instead of -eq

[ $a -eq $b ]     # ✓
(( a == b ))       # ✓ (Bash)

if [ "$file" = "*.sh" ]   # ✗ literal string comparison
if [[ "$file" == *.sh ]]   # ✓ glob pattern in [[ ]]

while read line; do (( count++ )); done < "f"   # ✓

grep "pattern" file   # ✗ exits if nothing matches (rc=1)
grep "pattern" file || true  # ✓

QUOTING

"$var" = Safe. Always use.
$var = Dangerous. Splits on IFS.

(( )) = Arithmetic only

while read; done < <(cmd) → stays in parent shell, vars preserved
